microfeed automatically generates three types of feeds from your content: a web feed (HTML), an RSS feed (XML), and a JSON feed. All three feeds stay in sync and update instantly when you publish new content.

## Web Feed

Your web feed is the main public-facing website, accessible at your custom domain or `your-project.pages.dev`.

## RSS Feed

The RSS feed is automatically generated and available at `/rss/` (e.g., `https://your-site.com/rss/`).

### RSS Generation

The RSS feed is built by `FeedPublicRssBuilder.js`:

```javascript
_buildItemsRss() {
  const items = [];
  this.jsonData.items.forEach((item) => {
    const _microfeed = item._microfeed || {};
    const itemJson = {
      'title': item.title || 'untitled',
      'guid': item.id,
      'pubDate': msToUtcString(item._microfeed.date_published_ms),
      'itunes:explicit': _microfeed['itunes:explicit'] ? 'true' : 'false',
    };
    
    if (item['content_html']) {
      itemJson['description'] = {
        '@cdata': item['content_html'],
      };
    }

    if (item['url']) {
      itemJson['link'] = item['url'];
    }

    // Add enclosure for media files
    const {attachments} = item;
    let mediaFile;
    if (attachments && attachments[0]) {
      mediaFile = attachments[0];
    }
    if (mediaFile && mediaFile.url) {
      itemJson.enclosure = {
        '@_url': mediaFile.url,
        '@_type': mediaFile.mime_type,
        '@_length': mediaFile.size_in_byte,
      };
      if (mediaFile.duration_in_seconds) {
        itemJson['itunes:duration'] = secondsToHHMMSS(mediaFile.duration_in_seconds);
      }
    }
    items.push(itemJson);
  });
  return items;
}
```

### iTunes Podcast Compatibility

The RSS feed includes full iTunes podcast namespace support:

- `itunes:type` - Episodic or serial
- `itunes:explicit` - Explicit content flag
- `itunes:category` - Podcast categories
- `itunes:owner` - Owner email and name
- `itunes:image` - Podcast artwork
- `itunes:season` - Season number (per episode)
- `itunes:episode` - Episode number (per episode)
- `itunes:episodeType` - Full, trailer, or bonus
- `itunes:duration` - Episode duration
- `itunes:block` - Block from iTunes
- `itunes:complete` - Mark podcast as complete

### RSS XML Structure

```xml
<?xml version='1.0' encoding='UTF-8'?>
<?xml-stylesheet href="/rss/stylesheet" type="text/xsl"?>
<rss xmlns:itunes='http://www.itunes.com/dtds/podcast-1.0.dtd'

     version='2.0'>
  <channel>
    <title>Your Feed Title</title>
    <language>en</language>
    <generator>microfeed.org</generator>
    <itunes:type>episodic</itunes:type>
    <itunes:explicit>false</itunes:explicit>
    <atom:link rel="self" href="https://your-site.com/rss/" type="application/rss+xml"/>
    <description><![CDATA[Your feed description]]></description>
    <itunes:image href="https://your-site.com/cover.jpg"/>
    
    <item>
      <title>Episode Title</title>
      <guid>uhbQEmArlC2</guid>

      <description><![CDATA[Episode description]]></description>

      <itunes:season>1</itunes:season>
      <itunes:episode>1</itunes:episode>
      <itunes:episodeType>full</itunes:episodeType>
    </item>
  </channel>
</rss>
```

### Pagination

The RSS feed supports pagination with `next_cursor` and `prev_cursor` parameters:

```xml
<link rel="next" 
      href="https://your-site.com/rss/?next_cursor=abc123&sort=newest_first" 
      type="application/rss+xml"/>
```

### JSON Feed Builder

The JSON feed is generated by `FeedPublicJsonBuilder.js`:

```javascript
_buildPublicContentItem(item, mediaFile) {
  const newItem = {
    id: item.id,
    title: item.title || 'untitled',
  };
  
  const attachment = {};
  const _microfeed = {
    is_audio: mediaFile.isAudio,
    is_document: mediaFile.isDocument,
    is_external_url: mediaFile.isExternalUrl,
    is_video: mediaFile.isVideo,
    is_image: mediaFile.isImage,
    web_url: item.webUrl,
    json_url: item.jsonUrl,
    rss_url: item.rssUrl,
    guid: item.guid,
    status: ITEM_STATUSES_DICT[item.status].name,
  };

  if (isValidMediaFile(mediaFile)) {
    if (mediaFile.url) {
      attachment['url'] = buildAudioUrlWithTracking(mediaFile.url, trackingUrls);
    }
    if (mediaFile.contentType) {
      attachment['mime_type'] = mediaFile.contentType;
    }
    if (mediaFile.sizeByte) {
      attachment['size_in_byte'] = mediaFile.sizeByte;
    }
    if (mediaFile.durationSecond) {
      attachment['duration_in_seconds'] = mediaFile.durationSecond;
      _microfeed['duration_hhmmss'] = secondsToHHMMSS(mediaFile.durationSecond);
    }
  }

  newItem['_microfeed'] = _microfeed;
  return newItem;
}
```

### Headless CMS Usage

The JSON feed can be used as a headless CMS API:

<Tabs>
  <Tab title="Fetch Feed Data">
    ```javascript
    // Fetch feed data
    const response = await fetch('https://your-site.com/json/');
    const feed = await response.json();

    // Access items
    feed.items.forEach(item => {
      console.log(item.title);
      console.log(item._microfeed.web_url);
      if (item._microfeed.is_audio) {
        console.log('Audio duration:', item._microfeed.duration_hhmmss);
    ```

  <Tab title="Pagination">

    ```javascript
    // Fetch first page
    let url = 'https://your-site.com/json/';

    const feed = await response.json();

    // Check for next page
    if (feed._microfeed.next_url) {
      const nextResponse = await fetch(feed._microfeed.next_url);
    ```

## OpenAPI Specification

microfeed provides an OpenAPI specification for the JSON feed API:

- **YAML:** `https://your-site.com/json/openapi.yaml`
- **HTML:** `https://your-site.com/json/openapi.html`

<Info>
The OpenAPI spec documents all available fields, query parameters, and response formats.
</Info>

## Feed Query Parameters

All feed types support these query parameters:

| Parameter | Description | Example |
|-----------|-------------|----------|
| `next_cursor` | Pagination cursor for next page | `?next_cursor=abc123` |
| `prev_cursor` | Pagination cursor for previous page | `?prev_cursor=xyz789` |
| `sort` | Sort order: `newest_first` or `oldest_first` | `?sort=oldest_first` |
