## R2 Setup

During installation, you configure R2 access with these credentials:

- **R2_ACCESS_KEY_ID** - API access key
- **R2_SECRET_ACCESS_KEY** - API secret key
- **R2 Bucket Name** - Your storage bucket (e.g., `microfeed-media`)
- **Public Bucket URL** - CDN URL for serving files (e.g., `https://media-cdn.microfeed.org`)

<Info>
Cloudflare R2 includes a generous free tier: 10GB storage + 10 million reads/month + 1 million writes/month.
</Info>

## File Upload Process

### From Admin Dashboard

1. Navigate to **Add new item** in the admin dashboard
2. Select content type (audio, video, image, or document)
3. Drag and drop your file or click to browse
4. File uploads automatically to R2
5. Progress indicator shows upload status
6. Metadata (duration, file size) is extracted automatically

### Upload Implementation

The upload process uses presigned URLs for secure, direct uploads to R2:

```javascript
onFileUpload(file) {
  const {category} = this.state;
  this.setState({uploadStatus: UPLOAD_STATUS__START});
  
  const {name, size, type} = file;
  const extension = name.slice((name.lastIndexOf('.') - 1 >>> 0) + 2);
  let newFilename = `${category}-${randomHex(32)}`;
  if (extension && extension.length > 0) {
    newFilename += `.${extension}`;
  }
  const cdnFilename = `media/${newFilename}`;

  Requests.upload(file, cdnFilename, (percentage) => {
    this.setState({progressText: `${parseFloat(percentage * 100.0).toFixed(2)}%`});
  }, (cdnUrl) => {
    this.setState({
      progressText: null,
      uploadStatus: null,
      url: cdnUrl,
      contentType: type,
      sizeByte: size,
    });
  });
}
```

## File Naming Convention

Uploaded files are automatically renamed with a secure pattern:

```
media/{content-type}-{random-hex-32}.{extension}
```

Examples:
- `media/audio-abc123def456789.mp3`
- `media/video-xyz789abc123456.mp4`
- `media/image-def456ghi789012.jpg`
- `media/document-ghi012jkl345678.pdf`

<Tip>
The random hex prefix prevents filename collisions and makes URLs unpredictable for better security.
</Tip>

## CDN Configuration

To serve files from R2, you need to configure a public bucket URL:

1. Create a custom subdomain (e.g., `media.your-site.com`)
2. Point it to your R2 bucket using Cloudflare DNS
3. Configure R2 public access
4. Set the **Public Bucket URL** in Settings > Web Global Settings

### URL Structure

Media URLs follow this pattern:

```
https://media-cdn.microfeed.org/media/audio-abc123def456.mp3
         ↑                        ↑
    Public Bucket URL          File Path
```

The URL builder concatenates these:

```javascript
if (item.image) {
  item.image = urlJoinWithRelative(this.publicBucketUrl, item.image);
}
```

## File Management Features

### Replace Media File

You can replace an existing media file:

1. Open the item in edit mode
2. Expand "Upload a new file to replace this one"
3. Upload the new file
4. Previous file remains in R2 but is no longer referenced

<Warning>
Replacing a file does not delete the old file from R2. You'll need to manually clean up unused files if storage becomes an issue.
</Warning>

### Delete Media File

To remove media from an item:

1. Change the content type (e.g., from audio to blog post)
2. Confirm the deletion
3. The media file reference is removed
4. File remains in R2 storage

### Download URLs

All media files are accessible via direct download URLs shown in the admin dashboard:

```html
<a href="https://media.your-site.com/media/audio-abc123.mp3" target="_blank">
  Download URL
</a>
```

## R2 Storage Limits

### Free Tier

- **Storage:** 10 GB
- **Class A Operations:** 1 million/month (writes)
- **Class B Operations:** 10 million/month (reads)

### Paid Tier (when exceeded)

- **Storage:** $0.015 per GB/month
- **Class A Operations:** $4.50 per million
- **Class B Operations:** $0.36 per million
- **Egress:** $0.00 (always free)

<Tip>
R2's zero egress fees make it ideal for podcasts and video hosting, where bandwidth costs can be significant with other providers.
</Tip>

## Backup and Data Export

To backup your media files from R2:

### Using S3-Compatible Tools

R2 is S3-compatible, so you can use tools like:

- AWS CLI with R2 endpoint
- rclone
- Cyberduck
- s3cmd

### Example with AWS CLI

```bash
# Configure AWS CLI with R2 credentials
aws configure --profile r2

# Sync R2 bucket to local directory
aws s3 sync \
  s3://your-bucket-name \
  ./local-backup \
  --endpoint-url https://[account-id].r2.cloudflarestorage.com \
  --profile r2
```

<Info>
See the [Cloudflare R2 documentation](https://developers.cloudflare.com/r2/data-access/s3-api/) for S3 API details.
</Info>

## Troubleshooting

### Upload Fails

- Check R2 credentials in GitHub secrets
- Verify R2 bucket exists and has correct permissions
- Ensure file size is within R2 limits (max 5TB per object)

### Files Not Accessible

- Verify Public Bucket URL is configured correctly
- Check R2 bucket has public access enabled
- Confirm custom domain DNS is pointing to R2

### Slow Downloads

- Enable Cloudflare CDN caching for your R2 bucket
- Use a custom domain with Cloudflare proxy enabled
- Compress media files to reduce size
