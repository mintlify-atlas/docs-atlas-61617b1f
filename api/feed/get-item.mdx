title: 'Get Item'

## Overview

The `/i/{id}/json` endpoint returns JSON Feed data for a single item. This is a public, read-only endpoint that doesn't require authentication.

<Info>
  The response follows the [JSON Feed 1.1 specification](https://www.jsonfeed.org/version/1.1/), with the `items` array containing only the requested item.
</Info>

## Endpoint

```
GET /i/{id}/json
```

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the item to fetch.
  
  **Example:** `GOwipulXxh9`
</ParamField>

## Response

Returns a JSON Feed object with channel metadata and a single item in the `items` array.

<ResponseField name="version" type="string" required>
  The JSON Feed version URL. Always `https://jsonfeed.org/version/1.1`
</ResponseField>

<ResponseField name="title" type="string" required>
  The channel name
</ResponseField>

<ResponseField name="description" type="string">
  Channel description in HTML format
</ResponseField>

<ResponseField name="home_page_url" type="string">
  The channel's website URL
</ResponseField>

<ResponseField name="feed_url" type="string">
  The URL of the main JSON feed
</ResponseField>

<ResponseField name="icon" type="string">
  Channel cover image URL
</ResponseField>

<ResponseField name="favicon" type="string">
  Favicon URL for the site
</ResponseField>

<ResponseField name="language" type="string">
  Primary language code in RFC 5646 format (e.g., `en-us`)
</ResponseField>

<ResponseField name="items" type="array">
  Array containing a single item object. See the Item Structure section below.
</ResponseField>

<ResponseField name="_microfeed" type="object">
  Microfeed-specific metadata (pagination fields are not included for single item responses)
</ResponseField>

### Item Structure

<ResponseField name="id" type="string" required>
  Unique item identifier
</ResponseField>

<ResponseField name="title" type="string" required>
  Item title (e.g., podcast episode title or blog post title)
</ResponseField>

<ResponseField name="content_html" type="string">
  Item description in HTML format. For podcast episodes, recommended to keep under 4000 characters for Apple Podcasts compatibility.
</ResponseField>

<ResponseField name="content_text" type="string">
  Plain text version of the description
</ResponseField>

<ResponseField name="url" type="string">
  Web page URL for this item on your microfeed site
</ResponseField>

<ResponseField name="external_url" type="string">
  External URL if the attachment is an external link
</ResponseField>

<ResponseField name="date_published" type="string">
  Publication date in RFC 3339 format (e.g., `2023-01-03T00:35:14.994Z`)
</ResponseField>

<ResponseField name="date_modified" type="string">
  Last modification date in RFC 3339 format
</ResponseField>

<ResponseField name="image" type="string">
  Square image specific to this item. For podcast episodes, must be 1400x1400 to 3000x3000 pixels (JPEG or PNG, 72 dpi, RGB colorspace)
</ResponseField>

<ResponseField name="banner_image" type="string">
  Banner image URL (only when attachment is an image)
</ResponseField>

<ResponseField name="language" type="string">
  Item-specific language code (if different from channel default)
</ResponseField>

<ResponseField name="attachments" type="array">
  Array of media attachments. Although this is an array, microfeed supports up to 1 attachment per item.
  
  Each attachment object contains:
  - `url` - Media file URL or external URL
  - `mime_type` - Content type (e.g., `audio/mpeg`, `video/mp4`, `image/jpeg`)
  - `size_in_bytes` - File size (not present for external URLs)
  - `duration_in_seconds` - Duration for audio/video files
</ResponseField>

<ResponseField name="_microfeed" type="object">
  Item-specific metadata including:
  
  **Status and Type:**
  - `status` - Publication status: `published`, `unlisted`, or `unpublished`
  - `is_audio` - Boolean indicating if attachment is audio
  - `is_video` - Boolean indicating if attachment is video
  - `is_image` - Boolean indicating if attachment is an image
  - `is_document` - Boolean indicating if attachment is a document
  - `is_external_url` - Boolean indicating if attachment is an external URL
  
  **URLs:**
  - `web_url` - Web page URL for this item
  - `json_url` - This endpoint's URL
  - `rss_url` - RSS version of this item
  
  **Metadata:**
  - `guid` - Unique identifier used by feed readers
  - `date_published_short` - Human-friendly date format
  - `date_published_ms` - Unix timestamp in milliseconds
  - `duration_hhmmss` - Duration in HH:MM:SS format (for audio/video)
  
  **iTunes/Podcast Fields:**
  - `itunes:title` - Episode title for Apple Podcasts
  - `itunes:block` - Hide from Apple Podcasts if `true`
  - `itunes:episodeType` - Episode type: `full`, `trailer`, or `bonus`
  - `itunes:season` - Season number (integer)
  - `itunes:episode` - Episode number (integer)
  - `itunes:explicit` - Explicit content flag
</ResponseField>

## Error Responses

### 404 Not Found

Returned when:
- The item with the specified ID doesn't exist
- The item exists but has `unpublished` status
- The JSON feed is disabled in admin settings at `/admin/settings/`

```bash
Not Found
```

## Use Cases

- **Direct Item Links**: Share individual items via feed readers
- **Item Previews**: Generate previews for social media or embeds
- **Selective Sync**: Sync specific items to external platforms
- **Feed Reader Support**: Compatible with JSON Feed readers
- **Content Distribution**: Distribute individual pieces of content

## Notes

- Items with `unlisted` status are accessible via this endpoint but don't appear in the main feed
- Items with `unpublished` status return a 404 error
- The response includes full channel metadata, not just the item data
- This endpoint can be disabled in admin settings along with the main JSON feed
