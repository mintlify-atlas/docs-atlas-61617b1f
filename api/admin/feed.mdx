<Warning>
  This endpoint requires authentication using your API key. You can find your API key in the API section of your microfeed settings at `/admin/settings/`.
</Warning>

## Get Feed

### `GET /api/feed/`

Fetch JSON feed data for your entire feed. This endpoint returns the same data as `GET /json` but requires authentication and cannot be disabled via settings.

This is useful when you need to:
- Access your feed programmatically even when public access is disabled
- Fetch unpublished or unlisted items in your scripts
- Build admin tools that need guaranteed access to feed data

#### Authentication

<ParamField header="X-MicrofeedAPI-Key" type="string" required>
  Your microfeed instance's API key. Find it at `/admin/settings/` in the API section.
</ParamField>

#### Query Parameters

<ParamField query="next_cursor" type="integer">
  Offset for pagination. Use the `next_cursor` value from a previous response to fetch the next page of items.
  
  Example: `1672707197212`
</ParamField>

<ParamField query="sort" type="string" default="newest_first">
  Sort order for items.
  
  Options:
  - `newest_first` - Most recent items first (default)
  - `oldest_first` - Oldest items first
</ParamField>

#### Response

Returns a JSON Feed object containing:
- Channel metadata (title, description, icon, etc.)
- Array of items with full metadata
- Pagination information in the `_microfeed` field

See the [JSON Feed specification](https://www.jsonfeed.org/version/1.1/) for full schema details.

<ResponseField name="version" type="string">
  JSON Feed version URL (always "https://jsonfeed.org/version/1.1")
</ResponseField>

<ResponseField name="title" type="string">
  Channel title
</ResponseField>

<ResponseField name="description" type="string">
  Channel description
</ResponseField>

<ResponseField name="home_page_url" type="string">
  Channel homepage URL
</ResponseField>

<ResponseField name="feed_url" type="string">
  JSON feed URL
</ResponseField>

<ResponseField name="items" type="array">
  Array of item objects. Each item includes title, content, attachments, and metadata.
</ResponseField>

<ResponseField name="_microfeed" type="object">
  microfeed-specific metadata including pagination cursors and item counts.
  
  <ResponseField name="_microfeed.items_next_cursor" type="integer">
    Cursor for the next page of items. Use this value in the `next_cursor` query parameter.
  </ResponseField>
  
  <ResponseField name="_microfeed.items_sort_order" type="string">
    Current sort order applied to items.
  </ResponseField>
</ResponseField>

#### Example Request

```bash
curl -X GET https://your-instance.microfeed.org/api/feed/ \
  -H "X-MicrofeedAPI-Key: your-api-key-here"
```

#### Example Request with Pagination

```bash
curl -X GET "https://your-instance.microfeed.org/api/feed/?next_cursor=1672707197212&sort=oldest_first" \
  -H "X-MicrofeedAPI-Key: your-api-key-here"
```

#### Error Responses

<ResponseField name="401 Unauthorized" type="error">
  Invalid or missing API key.
  
  ```json
  {
    "error": "Unauthorized"
  }
  ```
</ResponseField>

## Comparison with Public Feed API

| Feature | `GET /json` (Public) | `GET /api/feed/` (Admin) |
|---------|---------------------|-------------------------|
| Authentication | None required | API key required |
| Can be disabled | Yes, via settings | No |
| Access to unpublished items | No | Depends on implementation |
| Use case | Public consumption | Admin/automation |

## Use Cases

- **Automated backups**: Fetch all feed data regularly for backup purposes
- **Content migration**: Export content to other platforms
- **Admin dashboards**: Build custom admin interfaces that need feed data
- **Testing**: Test feed generation without public exposure
