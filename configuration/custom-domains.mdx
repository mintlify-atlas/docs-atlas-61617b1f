By default, your microfeed site is accessible at `{project-name}.pages.dev`. You can configure a custom domain to use your own domain name, such as `podcast.example.com` or `blog.example.com`.

## Prerequisites

Before setting up a custom domain:

1. **Own a domain name** - Register a domain through any registrar (Namecheap, Google Domains, etc.)
2. **Have DNS access** - Ability to modify DNS records for your domain
3. **Deployed microfeed** - Your microfeed instance is live at `{project-name}.pages.dev`

<Warning>
  You cannot use a root domain (like `example.com`) for Cloudflare Pages custom domains. You must use a subdomain (like `www.example.com` or `podcast.example.com`).
</Warning>

## Option 1: Domain on Cloudflare (Recommended)

If your domain's DNS is managed by Cloudflare, setup is automatic.

### Step 1: Add Domain to Pages Project

1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. Navigate to **Workers & Pages**
3. Select your microfeed project
4. Click the **Custom domains** tab
5. Click **Set up a custom domain**
6. Enter your subdomain (e.g., `podcast.example.com`)
7. Click **Continue**

### Step 2: Automatic DNS Configuration

Cloudflare automatically creates a CNAME record pointing to your Pages project:

```
podcast.example.com CNAME {project-name}.pages.dev
```

### Step 3: Verify

Wait 1-2 minutes, then:
- Visit your custom domain in a browser
- Verify it loads your microfeed site
- Check for HTTPS lock icon (SSL is automatic)

<Accordion title="DNS propagation time">
  Usually instant for Cloudflare-managed domains, but can take up to 24 hours in rare cases. Check status in the Custom domains tab.
</Accordion>

## Option 2: Domain on Another Registrar

If your domain's DNS is managed by another service (Namecheap, GoDaddy, etc.), you'll manually configure DNS.

### Step 1: Add Domain to Pages Project

1. Go to [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. Navigate to **Workers & Pages**
3. Select your microfeed project
4. Click the **Custom domains** tab
5. Click **Set up a custom domain**
6. Enter your subdomain (e.g., `blog.example.com`)
7. Click **Continue**

### Step 2: Get DNS Instructions

Cloudflare displays DNS records you need to create. You'll see something like:

```
Type: CNAME
Name: blog
Value: {project-name}.pages.dev
```

### Step 3: Add DNS Record at Your Registrar

**For Namecheap**:
1. Log into Namecheap
2. Go to Domain List > Manage
3. Advanced DNS tab
4. Add New Record
   - Type: CNAME Record
   - Host: `blog`
   - Value: `{project-name}.pages.dev`
   - TTL: Automatic
5. Save

**For GoDaddy**:
1. Log into GoDaddy
2. My Products > DNS
3. Add > CNAME
   - Name: `blog`
   - Value: `{project-name}.pages.dev`
   - TTL: 1 hour
4. Save

**For Other Registrars**:
Find your registrar's DNS management interface and create a CNAME record with:
- **Host/Name**: Your subdomain (e.g., `blog`, `podcast`)
- **Value/Target**: `{project-name}.pages.dev`
- **TTL**: Default or 3600 seconds

### Step 4: Verify DNS Propagation

Check if your DNS changes have propagated:

```bash
dig blog.example.com CNAME
```

Or use an online tool like [whatsmydns.net](https://www.whatsmydns.net/)

<Accordion title="How long does DNS propagation take?">
  - Typically: 5-30 minutes
  - Worst case: Up to 48 hours
  - Depends on: TTL values and DNS server caching
</Accordion>

### Step 5: Wait for SSL Certificate

After DNS propagates:
1. Cloudflare automatically provisions an SSL certificate
2. This takes 1-5 minutes
3. Your site becomes available at `https://blog.example.com`

## Multiple Custom Domains

You can add multiple custom domains to the same microfeed site:

1. Follow the setup process for each domain
2. Each domain shows the same content
3. Useful for:
   - Multiple languages (en.example.com, es.example.com)
   - Brand variations (podcast.com, podcast.net)
   - Redirecting old domains

## Using Root Domains (Advanced)

Cloudflare Pages officially requires subdomains, but you can use a root domain with a workaround.

<Warning>
  This approach requires all your domain's DNS to be managed by Cloudflare.
</Warning>

### Step 1: Transfer DNS to Cloudflare

1. Add your domain to Cloudflare
2. Update nameservers at your registrar
3. Wait for DNS to migrate (24-48 hours)

### Step 2: Create CNAME Flattening

1. In Cloudflare DNS settings
2. Add a CNAME record:
   - Name: `@` (represents root)
   - Target: `{project-name}.pages.dev`
   - Proxy status: Proxied (orange cloud)
3. Cloudflare automatically flattens this to an A record

### Step 3: Add to Pages Project

1. In Pages Custom domains
2. Add `example.com` (without subdomain)
3. Cloudflare validates the CNAME

<Accordion title="Why does this work?">
  Cloudflare's CNAME flattening feature converts CNAME records at the root level into A records, which are allowed at the root. Most DNS providers don't support this.
</Accordion>

## WWW Redirect

To redirect `www.example.com` to `example.com` (or vice versa):

### Step 1: Add Both Domains

Add both domains as custom domains in your Pages project:
- `example.com`
- `www.example.com`

### Step 2: Create Page Rule

1. Go to Cloudflare Dashboard > Rules > Page Rules
2. Create rule:
   - URL: `www.example.com/*`
   - Setting: Forwarding URL
   - Status Code: 301 (Permanent Redirect)
   - Destination: `https://example.com/$1`
3. Save and Deploy

Now `www.example.com` redirects to `example.com` automatically.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Custom domain shows 'DNS Not Found' error">
    **Cause**: DNS record not created or not propagated
    
    **Fix**:
    1. Verify CNAME record exists in your DNS settings
    2. Check spelling of subdomain and target
    3. Wait 30 minutes for propagation
    4. Use `dig` or online tools to verify DNS
    5. Try clearing your browser cache
  </Accordion>
  
  <Accordion title="Custom domain shows 'SSL Certificate Pending'">
    **Cause**: Cloudflare is provisioning your SSL certificate
    
    **Fix**:
    - Wait 5-15 minutes
    - Ensure DNS is properly configured
    - Check Custom domains tab for status
    - If stuck after 1 hour, remove and re-add the domain
  </Accordion>
  
  <Accordion title="Site works on pages.dev but not custom domain">
    **Cause**: DNS misconfiguration or not propagated
    
    **Fix**:
    1. Verify CNAME record points to `{project-name}.pages.dev`
    2. Check DNS propagation with online tools
    3. Try accessing from another device/network
    4. Clear browser cache and cookies
    5. Check for typos in domain configuration
  </Accordion>
  
  <Accordion title="Certificate shows as invalid or untrusted">
    **Cause**: Cloudflare hasn't issued certificate yet
    
    **Fix**:
    - Wait 5-10 minutes for certificate issuance
    - Ensure DNS is pointed correctly
    - Check that SSL/TLS mode is set to "Full" in Cloudflare
    - Remove and re-add custom domain if stuck
  </Accordion>
  
  <Accordion title="Old content showing after custom domain setup">
    **Cause**: Browser cache or DNS cache
    
    **Fix**:
    1. Hard refresh: Ctrl+F5 (Windows) or Cmd+Shift+R (Mac)
    2. Clear browser cache
    3. Try incognito/private browsing
    4. Flush DNS cache:
       - Windows: `ipconfig /flushdns`
       - Mac: `sudo dscacheutil -flushcache`
       - Linux: `sudo systemd-resolve --flush-caches`
  </Accordion>
  
  <Accordion title="'Too many redirects' error">
    **Cause**: SSL/TLS encryption mode mismatch
    
    **Fix**:
    1. Go to Cloudflare Dashboard
    2. Navigate to SSL/TLS > Overview
    3. Set encryption mode to "Full" or "Full (strict)"
    4. Wait 1-2 minutes
    5. Clear browser cache and retry
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Use meaningful subdomains">
    Choose subdomains that reflect your content:
    - `podcast.example.com` for podcasts
    - `blog.example.com` for blogs
    - `news.example.com` for news feeds
    - `updates.example.com` for changelogs
  </Accordion>
  
  <Accordion title="Enable HTTPS-only mode">
    After custom domain setup:
    1. Go to Cloudflare Dashboard > SSL/TLS > Edge Certificates
    2. Enable "Always Use HTTPS"
    3. This redirects all HTTP requests to HTTPS
  </Accordion>
  
  <Accordion title="Monitor custom domain status">
    Regularly check:
    - SSL certificate expiration (Cloudflare auto-renews)
    - DNS record configuration
    - Custom domain list in Pages dashboard
  </Accordion>
  
  <Accordion title="Update RSS feed links">
    After setting up custom domain:
    1. Update RSS feed submission to podcast directories
    2. Submit new URL to Apple Podcasts, Spotify, etc.
    3. Some directories support redirects, others require resubmission
  </Accordion>
  
  <Accordion title="Keep pages.dev domain as backup">
    Don't delete your `{project-name}.pages.dev` domain:
    - Useful for testing
    - Backup access if custom domain has issues
    - Used as CNAME target
  </Accordion>
</AccordionGroup>

## Updating Admin Dashboard Access

After setting up a custom domain, update how you access the admin dashboard.

### Before (using pages.dev):
```
https://{project-name}.pages.dev/admin/
```

### After (using custom domain):
```
https://podcast.example.com/admin/
```

<Warning>
  You'll need to reconfigure Cloudflare Zero Trust access if your admin URL changes. Update the application URL in your Zero Trust dashboard.
</Warning>

## Custom Domain for R2 Media Files

For optimal performance, also set up a custom domain for your R2 media files:

1. In Cloudflare Dashboard, go to **R2**
2. Select your bucket
3. Click **Settings** > **Custom Domains**
4. Add a custom domain (e.g., `media.example.com`)
5. Update **R2 Public Bucket URL** in microfeed Settings:
   ```
   https://media.example.com
   ```

This ensures your media files are served from your domain, not Cloudflare's R2 domain.
